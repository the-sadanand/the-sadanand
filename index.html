💡 About the Stats Cards

⚠ GitHub Camo Proxy Cache Fix

The public github-readme-stats instances are heavily rate-limited. When GitHub's Camo proxy gets a 504 Gateway Timeout or 429 Too Many Requests, it caches the failure until the TTL expires — this is called cache poisoning.

The README uses streak-stats.demolab.com (maintained by DenverCoder9 with dedicated infrastructure) and appends &cache_seconds=86400 to reduce re-fetches. If cards still break, hard-refresh (Ctrl+Shift+R) or wait ~1 hour for cache TTL.

The long-term fix is to self-host your own github-readme-stats instance on Vercel (free) and point the image URLs to your own deployment.
